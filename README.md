# Automation of Transport Services

**Automation of Transport Services** — настольное WPF-приложение для автоматизации работы организации, оказывающей транспортные услуги.

Система позволяет вести справочники клиентов, автомобилей, водителей и типов услуг, регистрировать выполненные перевозки, автоматически рассчитывать стоимость услуг и получать сводную статистику.

## Возможности

- управление клиентами;
- управление автомобилями;
- управление водителями;
- управление типами транспортных услуг;
- регистрация оказанных услуг;
- редактирование и удаление записей;
- поиск и фильтрация данных;
- автоматический расчёт стоимости поездки;
- просмотр истории услуг;
- формирование сводного документа;
- статистика по клиентам;
- статистика по автомобилям;
- статистика по водителям;
- статистика по типам услуг;
- работа с реляционной базой данных;
- автоматическое обновление интерфейса при изменении данных.

## Назначение проекта

Приложение предназначено для небольших транспортных компаний и диспетчерских служб, которым необходимо учитывать:

- заказчиков;
- используемые автомобили;
- работающих водителей;
- виды оказываемых услуг;
- расстояние поездок;
- стоимость километра;
- общую стоимость услуг;
- количество выполненных заказов.

## Архитектура

Приложение построено на многослойной архитектуре:

```text
Presentation
    │
    ▼
ViewModels
    │
    ▼
Services
    │
    ▼
Repositories
    │
    ▼
Entity Framework Core
    │
    ▼
Database
```

Основные слои:

### Presentation

Содержит WPF-представления, созданные на XAML.

Представления отвечают только за отображение данных и передачу пользовательских действий во ViewModel.

### ViewModels

ViewModel содержит:

- отображаемые данные;
- команды;
- обработку действий пользователя;
- открытие окон;
- загрузку справочников;
- проверку введённых значений;
- вызовы сервисного слоя.

Для синхронизации данных с интерфейсом используется `INotifyPropertyChanged`.

### Models

Модели используются в пользовательском интерфейсе и содержат данные, необходимые для привязки.

Основные модели:

- `ClientModel`;
- `DriverModel`;
- `VehicleModel`;
- `TypeOfServiceModel`;
- `StringOfServiceModel`.

### Services

Сервисный слой содержит бизнес-логику и связывает ViewModel с репозиториями.

Примеры сервисов:

- `IClientService`;
- `IDriverService`;
- `IVehicleService`;
- `IServiceTypeService`;
- `IServiceStringService`.

### Repositories

Репозитории отвечают за выполнение CRUD-операций и запросов к базе данных.

Они изолируют остальное приложение от деталей работы Entity Framework Core.

### Data Access

Слой доступа к данным включает:

- `DbContext`;
- сущности базы данных;
- конфигурацию связей;
- миграции;
- репозитории.

## Основные сущности

### Клиент

### Автомобиль

| Поле | Назначение |

| `Id` | Уникальный идентификатор |
| `Name` | Название или модель автомобиля |
| `PriceOfKm` | Стоимость одного километра |
| `Strings` | Список услуг с использованием автомобиля |

Содержит:

| Поле | Назначение |
|---|---|
| `Id` | Уникальный идентификатор |
| `Name` | ФИО водителя |
| `Strings` | Список выполненных услуг |

### Тип услуги

Содержит:

| Поле | Назначение |
|---|---|
| `Id` | Уникальный идентификатор |
| `Name` | Название типа услуги |
| `Strings` | Список услуг указанного типа |

### Оказанная услуга

Основная учётная запись системы.

Содержит:

| Поле | Назначение |
|---|---|

| `DriverId` | Водитель |
| `Cost` | Рассчитанная стоимость |

## Расчёт стоимости

Стоимость услуги рассчитывается по формуле:

```text
Стоимость = Расстояние × Цена одного километра
```

Пример:

```text
Расстояние: 150 км
Цена километра: 25 рублей

Стоимость = 150 × 25 = 3750 рублей
```

Расчёт выполняется автоматически на основании выбранного автомобиля.

## Связи между сущностями

```text
Client 1 ─────── N Service

Driver 1 ─────── N Service

Vehicle 1 ────── N Service

ServiceType 1 ── N Service
```

Каждая оказанная услуга связана с:

- одним клиентом;
- одним водителем;
- одним автомобилем;
- одним типом услуги.

При загрузке отдельных записей связанные сущности загружаются через `Include` и `ThenInclude`.

## Работа с услугами

- номер;
- дату;
- клиента;
- тип услуги;
- расстояние;

После выбора автомобиля система использует указанную для него стоимость километра и рассчитывает итоговую стоимость.


## Обновление интерфейса

Модели реализуют `INotifyPropertyChanged`.

При изменении свойства отправляется уведомление:

```csharp
PropertyChanged?.Invoke(
    this,
    new PropertyChangedEventArgs(propertyName)
);
```

Благодаря этому WPF автоматически обновляет связанные элементы интерфейса.

Клонируйте репозиторий:

## Проверки перед удалением

Удаление справочных записей должно учитывать наличие связанных услуг.

Рекомендуется не разрешать удаление:

- клиента, у которого есть услуги;
- автомобиля, использованного в услугах;
- водителя, связанного с услугами;
- типа услуги, который используется в документах.

Вместо физического удаления можно реализовать архивирование.
